# Claude

Reference these files for project context and rules:

- **GOALS.md** — What this project is trying to achieve and its design constraints
- **BACKLOG.md** — Implementation ideas and future work items
- **TESTLOG.md** — Invariants and success criteria that must never be violated
- **RULES.md** — Project-specific rules and quality standards that require human judgment
- **README.md** — Setup instructions and project structure

## Content rules

- **No em-dashes** — use `-` (space-hyphen-space) instead of `—`. Enforced by pre-commit hook.
- **Capitalization per language** — English uses Title Case for headings and project names. Latvian and French only capitalize the first word (e.g. "Ļoti labi treniņi", "Très bons entraînements").
